# Contributing Guidelines

## Getting Started
1.  Read `README.md` and `RULES.md` to understand the project structure.
2.  Pick a task from `PLAN.md` that is in the `ready` state.
3.  Assign yourself to the task.

## Development Workflow
1.  Create a branch following the naming convention: `feature/your-feature-name`.
2.  Write code and add tests.
3.  Run tests locally to ensure no regressions.
4.  Update `BITACORA.md` with your progress.

## Submitting Changes
1.  Push your branch.
2.  Open a Pull Request.
3.  Link the PR to the relevant task in `PLAN.md` (e.g., "Completes task-102").
4.  Wait for review and approval.
